# Promptlandia CLI Tools

This directory contains command-line interface (CLI) tools that expose the core functionality of Promptlandia. These scripts allow you to use Promptlandia's powerful prompt engineering features directly from your terminal, enabling integration into other workflows and scripts.

## Prerequisites

Ensure you have set up the project environment as described in the main [README](../README.md). You should have `uv` installed and a `.env` file configured with your GCP credentials.

## Available Tools

### 1. Prompt Trimmer (`cli/trim.py`)

Reduces prompt size by removing generic LLM best practices while preserving core task-specific instructions.

**Usage:**

```bash
uv run cli/trim.py [OPTIONS] [PROMPT]
```

**Arguments:**
*   `PROMPT`: The prompt to trim. If not provided as an argument, the script reads from standard input (stdin).

**Options:**
*   `-f FILE`, `--file FILE`: Read the prompt from a specific file.
*   `--show-analysis`: Output the intermediate analysis XML to stderr. This is useful for understanding *why* certain parts were removed.
*   `-v`, `--verbose`: Enable verbose logging to stderr.
*   `-h`, `--help`: Show the help message and exit.

**Examples:**

*   **Pipe from another command:**
    ```bash
    cat my_verbose_prompt.txt | uv run cli/trim.py
    ```

*   **Read from a file:**
    ```bash
    uv run cli/trim.py -f my_verbose_prompt.txt
    ```

*   **Pass directly as an argument:**
    ```bash
    uv run cli/trim.py "You are an expert... (lots of generic text) ... Write a poem about a cat."
    ```

*   **Show analysis:**
    ```bash
    echo "..." | uv run cli/trim.py --show-analysis
    ```

### 2. Prompt Improver (`cli/improve.py`)

Improves an existing prompt based on specific instructions, using a two-step "thinking" process to generate a high-quality revision.

**Usage:**

```bash
uv run cli/improve.py -i INSTRUCTIONS [OPTIONS] [PROMPT]
```

**Arguments:**
*   `PROMPT`: The prompt to improve. If not provided, reads from stdin.

**Options:**
*   `-i INSTRUCTIONS`, `--instructions INSTRUCTIONS` (**Required**): Specific instructions on how you want the prompt to be improved (e.g., "Make it more concise," "Add a persona," "Focus on step-by-step reasoning").
*   `-f FILE`, `--file FILE`: Read the prompt from a specific file.
*   `-s SYSTEM_PROMPT`, `--system-prompt SYSTEM_PROMPT`: An optional system prompt to include in the context.
*   `--show-plan`: Output the intermediate improvement plan generated by the model to stderr.
*   `-v`, `--verbose`: Enable verbose logging to stderr.
*   `-h`, `--help`: Show the help message and exit.

**Examples:**

*   **Basic improvement:**
    ```bash
    echo "Write a story about a dog." | uv run cli/improve.py -i "Make it a sci-fi story set on Mars."
    ```

*   **With a system prompt and showing the plan:**
    ```bash
    uv run cli/improve.py "Explain quantum computing." \
      -i "Target a 5th-grade audience and use analogies." \
      -s "You are a helpful science teacher." \
      --show-plan
    ```

### 3. Prompt Health Checklist (`cli/checklist.py`)

Evaluates a prompt against a comprehensive checklist of best practices and provides a detailed report on potential issues.

**Usage:**

```bash
uv run cli/checklist.py [OPTIONS] [PROMPT]
```

**Arguments:**
*   `PROMPT`: The prompt to evaluate. If not provided, reads from stdin.

**Options:**
*   `-f FILE`, `--file FILE`: Read the prompt from a specific file.
*   `--raw`: Output the raw markdown response from the model instead of the formatted report.
*   `-v`, `--verbose`: Enable verbose logging to stderr.
*   `-h`, `--help`: Show the help message and exit.

**Examples:**

*   **Evaluate a simple prompt:**
    ```bash
    echo "Write a poem about a cat." | uv run cli/checklist.py
    ```

*   **Evaluate a prompt from a file:**
    ```bash
    uv run cli/checklist.py -f my_prompt.txt
    ```
